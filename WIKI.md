## 1. ИДЕЯ И ЦЕЛЬ ПРОЕКТА

Telegram-бот для генерации профессиональных product-фотографий женского белья (трусы) для бренда **INTIMNO**.

**Задача бота:** помочь создавать красивые изображения трусов без фотографа — через нейросеть. Пользователь присылает фото своих трусов, бот генерирует готовое изображение в нужном стиле.

**Ключевое условие:** без фото трусов генерация не запускается. Трусы — основа каждого кадра.

**Исходный документ:** файл `Прототип — Telegram-бот NanoBanana.md` — там была первоначальная концепция, которая в процессе работы значительно расширилась.

## 3. КАК СТРОИЛСЯ БОТ: ЭТАПЫ

### Этап 1 — Первая версия (2 функции)

Первая версия бота включала только то, что было в прототипе:
- **Функция 1 `/reference`** — пользователь присылает референс → Claude анализирует → составляет промпт → пользователь присылает фото трусов → генерация → 1 изображение
- **Функция 2 `/style`** — пользователь присылает фото трусов → выбирает стиль (🌸 Нежный / 🌑 Тёмный) → 5 изображений

---

### Этап 2 — Исправление API-параметров

После получения реальной документации Replicate были исправлены параметры вызовов:

**NanaBanana Pro (до → после):**
```
было:   "width": 768, "height": 1024, "image": BytesIO(...)
стало:  "aspect_ratio": "3:4", "resolution": "1K",
        "output_format": "png", "safety_filter_level": "block_only_high",
        "image_input": [BytesIO(...)]   ← массив, не одиночное поле
```

**Claude Sonnet (до → после):**
```
было:   "prompt": system_prompt + user_prompt (всё в одно поле)
стало:  "system_prompt": ...,   ← отдельное поле
        "prompt": ...,
        "max_tokens": 4096,
        "extended_thinking": False
```

---

### Этап 3 — UX-улучшения (по запросу)

Добавлено по просьбе:

**Меню команд в Telegram:**  
При старте бот регистрирует команды через `bot.set_my_commands()` — они появляются в кнопке меню (иконка `/`) у пользователя.

**Приём изображений файлом:**  
Раньше бот принимал только сжатые фото из галереи. Добавлена поддержка файлов (PNG, JPG, JPEG, WEBP) — через `F.document` с проверкой MIME-типа.

**Отправка результата файлом:**  
Раньше бот отправлял сгенерированные изображения как сжатые фото (`answer_photo`). Это портило качество. Теперь бот скачивает PNG с Replicate через aiohttp и отправляет как документ (`answer_document`) — полное качество без сжатия Telegram.

---

### Этап 4 — Расширение функционала

**Функция 1 — 2 варианта вместо 1:**  
Claude теперь составляет 2 разных промпта по одному референсу. Запускаются 2 генерации параллельно → пользователь получает 2 варианта и выбирает лучший.

**Новая Функция 3 `/describe`:**  
Пользователь описывает желаемый стиль текстом (на русском или английском) → Claude составляет промпт → генерируется 1 изображение. Для тех, у кого нет референсов.

**Меню дополнительных элементов:**  
После получения фото трусов в любой функции бот предлагает добавить реквизит в кадр:

```
[📇 Визитка]         [📖 Журнал INTIMNO]
[✏️ Описать словами] [🖼 Прислать фото]
[✅ Без добавок — генерировать]
```

- **Визитка** → добавляет в промпт описание визитки с брендом INTIMNO
- **Журнал INTIMNO** → добавляет журнал в промпт
- **Описать словами** → пользователь пишет что добавить, текст вставляется в промпт
- **Прислать фото** → Claude анализирует фото реквизита и описывает его для промпта
- **Без добавок** → генерация запускается сразу

### Этап 5 — Обновление стилей

**Количество изображений:** уменьшено с 5 до **3** для каждого стиля.

**Добавлены 2 новых стиля:**

| Стиль | Характер |
|-------|---------|
| 💎 Rich | Белый шёлк, мрамор, бархат, золото, жемчуг, журнал INTIMNO, luxury editorial |
| 🎲 Смешанное | Неожиданные сочетания: диван, ноты, ромашки, пушистый плед, свечи |

Итого стилей: **4** (Нежный, Тёмный, Rich, Смешанное), по **3 промпта** каждый.

---

### Этап 7 — Динамическая генерация промптов для стилей

**Проблема:** в `/style` промпты были фиксированными — 12 текстов, прописанных вручную. Каждый раз при одном и том же стиле генерировались одинаковые кадры.

**Решение:** промпты теперь генерирует Claude на лету перед каждой съёмкой.

**Как это работает:**

1. Вместо фиксированных промптов в `prompts.py` — 4 шаблона-описания стиля (`STYLE_BLUEPRINTS`). Каждый описывает характеристики стиля: поверхности, реквизит, освещение, настроение, углы съёмки.
2. В `services.py` добавлена функция `generate_style_prompts(style_description)` — отправляет шаблон в Claude и получает 3 уникальных промпта.
3. В `bot.py` в момент выбора стиля бот показывает `⏳ Подбираю идеи для съёмки...`, вызывает Claude, затем продолжает обычный флоу с меню реквизита.

**Изменённые файлы:** `prompts.py`, `services.py`, `bot.py`

**Результат:** каждая генерация уникальна — разные поверхности, реквизит, углы — но всегда в рамках выбранного стиля.

## 4. ЧТО БЫЛО ИСПРАВЛЕНО ПО ХОДУ

### Критические исправления по подсказкам

**1. Неправильные параметры API NanaBanana Pro**  
Изначально использовались `width`/`height` и поле `image`. После того как была получена реальная документация API, исправлено на `aspect_ratio`, `resolution`, `image_input` (массив).

**2. Токены попали в `.env.example`**  
При создании файла `.env.example` в него случайно попали реальные токены. GitHub заблокировал пуш через встроенную защиту (Push Protection / Secret Scanning). Файл был исправлен (заменён на плейсхолдеры), коммит переделан.

**3. Стили: 5 изображений → 3**  
Изначально по прототипу было 5 промптов/изображений на стиль. По запросу изменено на 3 — более разумное количество.

**4. Два стиля → четыре**  
Прототип описывал только Нежный и Тёмный. Добавлены Rich и Смешанное с полными наборами промптов.

**5. Название сервиса в сообщениях**  
Бот писал "Отправляю в NanaBanana Pro" в каждом промежуточном сообщении. По просьбе убрано — оставлено только в `/start`, и написание исправлено на Nanobanana Pro.

## 5. ИТОГОВЫЙ ФУНКЦИОНАЛ

### Команды

| Команда | Описание | Результат |
|---------|----------|-----------|
| `/start` | Приветствие и список команд | — |
| `/help` | Подробная справка | — |
| `/reference` | Генерация по референс-изображению | 2 PNG |
| `/style` | Генерация по стилю из каталога | 3 PNG |
| `/describe` | Генерация по текстовому описанию стиля | 1 PNG |
| `/styles` | Описание всех 4 стилей | — |
| `/cancel` | Отменить текущее действие | — |

### Машина состояний

```
ReferenceFlow: waiting_reference → waiting_panties → choosing_extras → waiting_extras_text / waiting_extras_image
StyleFlow:     waiting_panties → choosing_style → choosing_extras → waiting_extras_text / waiting_extras_image
DescribeFlow:  waiting_description → waiting_panties → choosing_extras → waiting_extras_text / waiting_extras_image
```

### Форматы изображений

- **Принимает:** фото из галереи ИЛИ файл (PNG, JPG, JPEG, WEBP)
- **Отдаёт:** PNG-файл в полном качестве (без сжатия Telegram)

## 7. GITHUB И ДЕПЛОЙ НА СЕРВЕР

### Репозиторий

`github.com/oreobra/image-creator-INTIMNO`

### SSH-ключи

При деплое были созданы два SSH-ключа:
- **Mac → GitHub:** ключ сгенерирован на Mac, добавлен в GitHub Settings
- **Сервер → GitHub:** ключ сгенерирован на сервере командой `ssh-keygen`, добавлен в GitHub Settings

### Первый деплой на сервер

```bash
# На сервере — один раз:
ssh-keygen -t ed25519 -C "server" -f ~/.ssh/id_ed25519 -N ""
# добавить публичный ключ в github.com/settings/ssh/new

git clone git@github.com:oreobra/image-creator-INTIMNO.git bot
cd bot

nano .env          # вставить TELEGRAM_TOKEN и REPLICATE_API_TOKEN
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
newgrp docker
docker compose up -d
```

### Проверка работы

```bash
docker compose logs -f    # логи в реальном времени
```
